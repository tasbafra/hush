## 🐛 ПРОБЛЕМЫ КОТОРЫЕ БЫЛИ ИСПРАВЛЕНЫ

### 1. ✅ Дублирование мест в маршруте
**Проблема:** Одно и то же место появлялось несколько раз в маршруте

**Причина:** Функция `_select_route_candidates` не правильно проверяла уникальность ID

**Решение:**
- Добавлена строгая проверка `if not place_id or place_id in selected_ids: return False`
- Убедился что `selected_ids.add(place_id)` вызывается сразу после добавления
- Изменил `MIN_STOPS` с 3 до 2 для гибкости

**Файл:** `scoring.py`

**Тест:**
```python
ids = [s['id'] for s in result['stops']]
len(ids) == len(set(ids))  # True ✅
```

### 2. ✅ Расчёт времени маршрута
**Проблема:** При выборе "30 минут" предлагалось 4 места по 90 минут (итого 360 минут!)

**Причина:** 
- `normalize_profile` не сохранял `duration`
- `_select_route_candidates` не использовал время при выборе мест

**Решение:**
1. Добавил сохранение `duration` в `normalize_profile`
2. Добавил `duration_limit` в `_select_route_candidates`
3. Адаптивный `min_stops` в зависимости от времени:
   - 30min → 1 место
   - 1h → 2 места
   - 2h+ → 3+ места

**Файл:** `scoring.py`

**Тест:**
```
30 минут → 1 место, 30 мин ✅
1 час → 2 места, 60 мин ✅
2 часа → 3-4 места, 120 мин ✅
```

---

### 3. ✅ Замена мест в чате
**Проблема:** Кнопки "✕ [место]" не работали

**Причина:** Backend не обрабатывал запросы на замену мест

**Решение:**
1. Добавил обработку "не хочу", "замени", "не пойду" в `assistant_chat`
2. Извлечение названия места из сообщения
3. Вызов `replan_route_logic` с `action="replace_place:{place_id}"`
4. Функция `_handle_replace_place` в `replanner.py`

**Файлы:** `backend/main.py`, `backend/replanner.py`, `Web/src/AssistantChat.tsx`

**Пример диалога:**
```
User: "я не хочу идти в 'Успенская церковь'"
AI: "Заменили 'Успенская церковь' на 'Музей истории'."
```
